Tickers

Click a chip to toggle. Type above to search CoinGecko — pick a result to fetch 1y of daily prices and append it to the dataset. Every cell re-runs against the new universe.

1 polars · sql Per-ticker summary

One row per selected ticker — date range, row count, lowest / highest close, and the period-over-period change. Edit the SQL to drill into individual rows; e.g. SELECT * FROM prices ORDER BY date DESC LIMIT 20.

2 polars · sql Annualized return, volatility, Sharpe

Crypto markets trade 24/7, so we annualize by 365 (not 252 like stocks). The Sharpe ratio is excess return per unit of volatility — assuming a 0% risk-free rate (a fair simplification for crypto where stablecoin yields are noisy). daily_return is precomputed at load time and the first row of each coin (no prior close) is omitted from the dataset, so there's no NaN to filter here.

3 js · drawdown Max drawdown & days underwater

The deepest peak-to-trough loss in the period, plus how long the coin has been "underwater" (days since its all-time high). Polars- wasm doesn't support window functions yet, so we compute the running max + drawdown in JS, then surface the result as a cell output. This is the metric that actually decides whether a coin is investable for most allocators.

4 js · correlation Pairwise correlation matrix

Crypto correlations are notoriously high — most alts move with Bitcoin most of the time. The heatmap below is Pearson correlation of daily returns; cool colors = decoupled, warm = co-moving. Diversification is whatever is closest to zero with everyone else.

5 js · k-means Cluster by [return, vol, Sharpe, max_dd]

Four-feature K-means (k=3) over standardized [annualized return, volatility, Sharpe, max drawdown]. Labels Aggressive / Balanced / Defensive are sorted by mean Sharpe descending so they're stable across re-runs.

6 d3 · scatter Risk vs return, colored by cluster
7 d3 · equity curves Cumulative return — $1 invested over the period

Each line is what $1 invested at day 0 would be worth on each subsequent day, log-scaled so a 10× and a 100× both render. Hover to see the underlying coin and the exact multiple.
